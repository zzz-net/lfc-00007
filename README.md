# 值班交接白板

一个本地运行的运维班组值班交接白板系统，支持事项管理、交接班确认、历史记录查询等功能。

## 功能特性

- 📋 **看板式管理**：五列看板（新建/处理中/待接班确认/已接收/已关闭）直观展示事项状态
- ⚙️ **灵活配置**：支持班次、角色、用户、交接模板和检查项的自定义配置
- 🔄 **事项流转**：完整的状态流转流程，每次操作都有历史记录
- 🛡️ **权限控制**：基于角色的权限管理，不同角色有不同操作权限
- 📝 **版本控制**：乐观锁机制，两次基于同一版本的编辑会提示冲突
- ✅ **必填校验**：缺少必填检查项时不能进行接班确认
- 📤 **交接单导出**：支持导出文本格式的交接单
- 📜 **历史记录**：支持按班次、日期范围筛选历史交接记录
- 💾 **本地持久化**：使用 localStorage 存储，重启后数据不丢失

## 数据模型

### 班次 (Shift)
- id: 唯一标识
- name: 班次名称
- startTime: 开始时间 (HH:mm)
- endTime: 结束时间 (HH:mm)

### 角色 (Role)
- id: 唯一标识
- name: 角色名称
- canCreate: 可创建事项
- canProcess: 可处理事项
- canClose: 可关闭事项
- canConfirm: 可接班确认

### 用户 (User)
- id: 唯一标识
- name: 用户姓名
- roleId: 所属角色ID

### 检查项 (CheckItem)
- id: 唯一标识
- name: 检查项名称
- required: 是否必填
- description: 检查说明

### 事项 (Item)
- id: 唯一标识
- title: 事项标题
- type: 类型（alert告警/change变更/other其他）
- description: 事项描述
- status: 状态（new/processing/pending_confirm/received/closed）
- version: 版本号（用于冲突检测）
- shiftId: 所属班次
- shiftDate: 班次日期
- creatorId/creatorName: 创建人
- assigneeId/assigneeName: 处理人
- createTime/updateTime/closeTime: 时间戳
- closeReason: 关闭原因
- history: 操作历史数组

### 交接记录 (HandoverRecord)
- id: 唯一标识
- shiftId/shiftName: 班次信息
- date: 交接日期
- handoverUserId/handoverName: 交班人
- takeoverUserId/takeoverName: 接班人
- handoverTime/takeoverTime: 时间
- status: 状态（pending/confirmed）
- itemIds/itemCount: 关联事项
- checkResults: 检查项结果
- itemsSnapshot: 事项快照

4. 执行以下命令手动模拟冲突场景：
   ```javascript
   const item = Store.Items.getAll()[0];
   console.log('当前版本:', item.version);
   const result = Store.Items.update(item.id, { title: item.title + ' (其他人修改)' }, {id:'user_li',name:'李四'}, item.version);
   console.log('李四修改结果:', result.success, '新版本:', result.item?.version);
   ```

## 核心技术实现

### 数据持久化
使用浏览器 `localStorage` 进行数据存储，所有数据以 JSON 格式序列化存储。

### 版本冲突检测
采用乐观锁机制，每次修改时携带基础版本号，如果与当前版本不符则拒绝修改并返回冲突信息。

### 权限控制
基于角色的访问控制（RBAC），每个角色有独立的权限标志，操作前校验当前用户角色是否具备相应权限。

### 状态流转
事项状态流转图：
```
新建 → 处理中 → 待接班确认 → 已接收
 ↓        ↓          ↓          ↓
  --------- 可随时关闭 --------→ 已关闭
```

## 常见问题

**Q: 数据存储在哪里？会丢失吗？**
A: 数据存储在浏览器的 localStorage 中。只要不清除浏览器数据，数据就会一直保留。

**Q: 可以在多台设备间同步数据吗？**
A: 当前版本仅支持本地存储，不支持多设备同步。如需同步功能，需要后端服务支持。

**Q: 如何备份数据？**
A: 可以通过浏览器开发者工具导出 localStorage，或使用导出交接单功能导出重要数据。

**Q: 支持多用户同时在线编辑吗？**
A: 不支持，本应用是单机版应用。但内置了版本冲突检测机制，防止同一事项被覆盖修改。
